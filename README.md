# AI-помощник среднего чека Shamrock

MVP помогает бармену не изучать аналитику вручную. Он находит гостя в Telegram, выбирает контекст визита и получает одну короткую подсказку: что предложить, почему и какой фразой. После разговора бармен нажимает результат, а владелец видит допродажи в web-панели.

## Статическая версия для Vercel

В проект добавлена первая статическая версия без backend, базы, Telegram-бота и API-ключей:

- `index.html` — готовая демо-панель владельца.
- `assets/static.css` — внешний вид статической панели.
- `assets/static-app.js` — маленькая демо-логика: кнопки результата меняют цифры прямо в браузере.
- `vercel.json` — настройки, чтобы Vercel открывал эту страницу, в том числе по адресу `/admin`.
- `.vercelignore` — список локальных файлов, которые не нужно отправлять на Vercel.

Локальная проверка статической версии:

```powershell
python -m http.server 8080
```

После запуска открой: [http://127.0.0.1:8080](http://127.0.0.1:8080).

Для деплоя на Vercel достаточно загрузить проект в GitHub и импортировать его в Vercel как статический сайт. Build command можно оставить пустым. Output directory тоже можно оставить пустым, потому что `index.html` лежит в корне проекта.

## Что реализовано

- Telegram-меню: поиск и добавление гостя, рекомендации за день, личные результаты.
- Карточка гостя по имени, телефону или `loyalty_id`.
- Выбор контекста: один, двое, компания 3+.
- Rule-based механизм с оценкой вкусов, истории, размера компании, цены, маржи, приоритета и недавних отказов.
- Основная и дополнительная рекомендованные позиции.
- Пять результатов: предложил, купил, отказался, не предложил, неуместно.
- SQLite-база гостей, товаров, покупок, барменов и рекомендаций.
- Панель владельца `/admin` с дневной воронкой, дополнительной выручкой, лучшим барменом и лучшей позицией.
- Разделы гостей, товаров, рекомендаций и барменов.
- Начальные данные Shamrock: 10 товаров, 5 гостей, 2 бармена и история покупок.
- API для создания гостя, генерации рекомендации и фиксации результата.
- Автоматические тесты рекомендательного механизма, API, seed data, Telegram-форматов и web-панели.

## 1. Установка

Нужен Python 3.11 или новее. В папке проекта выполни:

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install -r requirements.txt
```

На macOS/Linux команда активации: `source .venv/bin/activate`.

## 2. Настройка `.env`

Создай рядом с `.env.example` файл `.env`:

```env
BOT_TOKEN=токен_от_BotFather
ADMIN_SECRET=
DATABASE_URL=sqlite:///./shamrock.db
BUSINESS_TIMEZONE=Asia/Yekaterinburg
```

Токен Telegram нельзя отправлять в чат или добавлять в публичный репозиторий.

`ADMIN_SECRET` зарезервирован для следующей версии. В текущем MVP панель `/admin` не имеет входа по паролю, поэтому запускай её только на своём компьютере и не открывай порт `8000` в интернет.

## 3. Подготовка тестовых данных

```powershell
python -m app.seed
```

Команду можно запускать повторно: товары и гости не дублируются.

## 4. Запуск web-панели

```powershell
uvicorn app.main:app --reload
```

Открой в браузере: [http://127.0.0.1:8000/admin](http://127.0.0.1:8000/admin).

Альтернативный запуск:

```powershell
python run.py
```

## 5. Запуск Telegram-бота

В отдельном окне с активированным окружением:

```powershell
python -m app.bot.bot
```

Если `BOT_TOKEN` пустой, бот остановится с понятным сообщением и не запустится.

## 6. Проверка рекомендаций и всего проекта

Только рекомендательный механизм:

```powershell
python -m pytest tests/test_recommendation_engine.py -v
```

Все проверки:

```powershell
python -m pytest -v
```

Проверка качества кода и зависимостей:

```powershell
python -m ruff check app tests run.py
python -m pip check
```

## 7. API

- `POST /api/guests` — создать гостя.
- `POST /api/recommendations/generate` — получить и сохранить рекомендацию.
- `POST /api/recommendations/{id}/status` — записать результат.
- `/docs` — интерактивное описание API.

## Как проверить MVP с барменом

1. Запусти seed, web-панель и Telegram-бота.
2. Попроси бармена найти тестового гостя «Алексей».
3. Выбери «Один» и проверь рекомендацию к IPA.
4. Нажми «Купил» или другой результат.
5. Обнови `/admin`: рекомендация и допродажа появятся в показателях.
6. Проведи пилот 7–14 смен и сравни средний чек, продажи кухни и конверсию рекомендаций.

## Ограничения MVP

Камер, распознавания лиц, сложной CRM и автоматической интеграции с FusionPOS пока нет. Панель не имеет авторизации, Bootstrap загружается из интернета, а SQLite подходит только для одного локального экземпляра приложения. Текущая версия предназначена для пилота и проверки главной гипотезы: увеличивает ли короткая персональная подсказка фактический средний чек.
